حزمة تشغيل مشروع Car History

المحتوى:
- نظرة عامة
- المتطلبات الأساسية
- إعداد الباكند (Django)
- إعداد الفرونتند (React + Vite)
- تشغيل المشروع في وضع التطوير
- تشغيل البناء للإنتاج
- ملاحظات مهمة

1. نظرة عامة
----------------
المشروع يتكون من جزئين رئيسيين:
- backend/ : تطبيق Django مع واجهة برمجة تطبيقات REST وواجهة إدارة.
- frontend/ : تطبيق React يستخدم Vite كما بنية تطويرية.

2. المتطلبات الأساسية
----------------
- Python 3.x مثبت.
- Node.js مثبت.
- pnpm منصب أو يمكنك استخدام npm/yarn لكن pnpm هو الموصى به لأن المشروع يستخدم pnpm.
- Git (اختياري).

3. إعداد الباكند (Django)
----------------
من داخل مجلد المشروع الرئيسي قم بالتالي:


3.1. إنشاء وتفعيل البيئة الافتراضية (إذا لم تكن موجودة)
- Windows PowerShell:
  python -m venv venv
  .\venv\Scripts\Activate.ps1

3.2. تحديث pip وتثبيت المتطلبات
  python -m pip install --upgrade pip
  cd backend
  python -m pip install -r requirements.txt

3.3. التأكد من ملف .env
- يوجد ملف backend/.env يستخدمه Django لقراءة المتغيرات التالية:
  SECRET_KEY
  DEBUG
  ALLOWED_HOSTS
- إذا لم يكن موجودًا، فعليك إنشاؤه بنفس الاسم وملء القيم.
- الوضع المحلي عادةً:
  DEBUG=True
  ALLOWED_HOSTS=localhost,127.0.0.1

3.4. إنشاء قاعدة البيانات وتطبيق الترحيلات
  python manage.py migrate

3.5. إنشاء مستخدم مسؤول (اختياري)
  python manage.py createsuperuser

4. إعداد الفرونتند (React + Vite)
----------------
4.1. الانتقال إلى مجلد الواجهة الأمامية
  cd ..\frontend

4.2. تثبيت حزم الواجهة الأمامية
  pnpm install

4.3. ملاحظة مهمة
- واجهة Vite تستخدم بروكسي لتوجيه طلبات /api و /admin إلى الخادم الخلفي
  target: http://localhost:8000
- لذلك يجب تشغيل خادم Django على المنفذ 8000 قبل أو أثناء تشغيل الواجهة الأمامية.

5. تشغيل المشروع في وضع التطوير
----------------
5.1. تشغيل خادم Django
  cd backend
  python manage.py runserver

5.2. تشغيل واجهة Vite
  cd ..\frontend
  pnpm dev

5.3. فتح المتصفح
- الواجهة الأمامية عادة على: http://localhost:3000
- إدارة Django متاحة عبر: http://localhost:3000/admin (أو http://localhost:8000/admin مباشرة)

6. تشغيل البناء للإنتاج
----------------
6.1. بناء الواجهة الأمامية
  cd frontend
  pnpm build

6.2. تجميع ملفات الستاتيك في Django
  cd ..\backend
  python manage.py collectstatic --noinput

6.3. تشغيل الخادم الثابت (اختياري)
- المشروع يحتوي أيضاً على خادم Express في frontend/server/index.ts
- لتشغيل النسخة المبنية بعد البناء:
  cd frontend
  pnpm start
- هذا سيشغل خادمًا على http://localhost:3000 ويقدم الملفات المبنية.

7. ملاحظات مهمة
----------------
- قاعدة البيانات الافتراضية هي SQLite وتُخزن في backend/db.sqlite3.
- إذا كنت تستخدم إعدادات محلية، فتأكد من أن ALLOWED_HOSTS صحيح.
- لا تنشر مفتاح SECRET_KEY أو مفاتيح API في أماكن عامة.
- في حال واجهت مشكلة في الحزم الأمامية، تأكد من أن إصدار pnpm مطابقة أو أحدث من 10.
- إذا أردت إعادة تثبيت الحزم من الصفر:
  pnpm install --frozen-lockfile

هذا الملف يشرح الخطوات العامة لتشغيل المشروع محليًا. إذا احتجت تعليمات إضافية حول نشر الإنتاج أو إعداد قواعد بيانات خارجية، يمكن إضافة فصل جديد لاحقًا.
